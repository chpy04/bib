# Plan: Replace Agent-based scraping with CodeAgent for deterministic scripts

## Context

BiB currently uses Browser Use `Agent` for both verification (discovering how to scrape) and runtime execution (fetching fresh data). Every data fetch spins up a full LLM-driven agent — slow, expensive, non-deterministic.

Browser Use provides `CodeAgent` with built-in session exporting (`session_to_python_script`, `export_to_ipynb`). It writes executable Python/JS code that can be saved and rerun deterministically without any LLM.

**Goal:** Replace the verification step with `CodeAgent` so it produces a reusable Python script. At runtime, execute that saved script directly (no LLM) instead of spinning up another agent. Fall back to re-running CodeAgent if the script fails.

## Current Codebase State (post-diff)

The codebase has already been refactored with a **profiles + dashboards** system:

- **Profiles** are named (e.g. `calm-brook`) instead of UUIDs, generated by `backend/app/names.py`
- **Profile metadata** stored in `profiles/{name}/meta.json` (url, site_name, created_at)
- **Dashboard persistence** in `profiles/{name}/dashboard.json` (component_code, verified_tasks, layout_hint, chat_history, url, prompt)
- **New routes**: `GET /api/profiles`, `GET /api/profiles/{id}`, `POST /api/profiles/{id}/tools` in `backend/app/routes/profiles.py`
- **New routes**: `GET /api/dashboards`, `GET /api/dashboards/{id}` in `backend/app/routes/data.py`
- **Frontend**: Profile grid view, profile detail view, dashboard loading from saved state
- **`GenerateUIRequest`** and **`RefineUIRequest`** now include `profile_id` (and url/prompt for generate)
- The generate and refine endpoints save/update `dashboard.json` automatically
- Frontend types extended with `ProfileSummary`, `ProfileDetail`, `DashboardSummary`, `DashboardDetail`

**agent.py is unchanged** from the original — still uses `Agent` for both `verify_task()` and `run_instruction()`. This is where the CodeAgent work goes.

## Architecture Change

```
BEFORE (current):
  Verify: Agent (LLM) → instructions (text steps) + sample_output
  Runtime: Agent (LLM) → follows text instructions → data

AFTER:
  Verify: CodeAgent (LLM) → executable Python script + sample_output
  Runtime: exec() the saved script → data (no LLM!)
  Fallback: if script fails → re-run CodeAgent → update saved script
```

## Changes

### 1. Rewrite `verify_task()` in `backend/app/agent.py`

Replace `Agent` with `CodeAgent`. Export the session to a Python script via `session_to_python_script`. The `instructions` field on `VerifiedTask` becomes the full script text instead of step-by-step text.

```python
from browser_use import CodeAgent
from browser_use.code_use.notebook_export import session_to_python_script

async def verify_task(task: Task, url: str) -> VerifiedTask | None:
    schema_json = json.dumps(task.output_schema, indent=2)
    task_prompt = (
        f"Navigate to {url} and perform the following task:\n"
        f"{task.description}\n\n"
        f"Extract the data and store it in a variable called `result` as a Python dict/list "
        f"matching this JSON schema:\n{schema_json}\n\n"
        f"The final result variable must be JSON-serializable."
    )

    browser = await _make_browser()
    try:
        agent = CodeAgent(
            task=task_prompt,
            llm=_make_browser_use_llm(),
            browser=browser,
            max_steps=20,
        )
        session = await agent.run()
    finally:
        await browser.stop()

    script = session_to_python_script(agent)
    sample_output = _extract_result_from_session(session)

    return VerifiedTask(
        id=task.id,
        description=task.description,
        output_schema=task.output_schema,
        display_hint=task.display_hint,
        type=task.type,
        instructions=script,  # full Python script, not text steps
        sample_output=sample_output,
    )
```

### 2. Rewrite `run_instruction()` in `backend/app/agent.py`

Replace agent-based runtime with direct script execution + fallback:

```python
async def run_instruction(instruction_name: str, profile_id: str) -> dict:
    instruction = get_instruction(profile_id, instruction_name)
    script = instruction["instructions"]  # Python script

    try:
        data = await _execute_script(script, profile_id)
        return {"instruction_name": instruction_name, "data": data, "success": True}
    except Exception as e:
        logger.warning("Script failed for '%s': %s. Falling back to CodeAgent.", instruction_name, e)
        # TODO: self-healing fallback — re-run CodeAgent, update registry
        return {"instruction_name": instruction_name, "data": None, "success": False, "error": str(e)}
```

### 3. Add `_execute_script()` helper in `backend/app/agent.py`

Runs the saved script in an isolated namespace with a browser page injected:

```python
async def _execute_script(script: str, profile_id: str) -> Any:
    browser = await _make_browser()
    try:
        # Inject browser/page into the script's namespace
        namespace = {"page": page, "browser": browser, "result": None}
        exec(compile(script, "<script>", "exec"), namespace)
        return namespace["result"]
    finally:
        await browser.stop()
```

> **Note:** Exact execution mechanism depends on `session_to_python_script` output format. May need async exec wrapper. Verify during implementation.

## Risks and open questions

1. **`session_to_python_script` output format** — Need to inspect actual output to finalize `_execute_script()`. May include boilerplate (imports, browser setup) we need to strip or adapt.

2. **Async execution** — Exported script likely contains `await` calls. Need async exec wrapper.

3. **Script sandboxing** — `exec()` on generated code has security implications. Acceptable for hackathon MVP; production would need subprocess/Docker sandboxing.

4. **Auth state** — Scripts need the authenticated browser session. `_make_browser()` already handles this via `storage_state`, so passing browser/page to the script namespace should work.

5. **`_make_browser_use_llm()` vs CodeAgent default** — CodeAgent defaults to `ChatBrowserUse()` if no llm provided. May be able to simplify by not passing llm at all.
